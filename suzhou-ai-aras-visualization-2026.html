📊 ARAS 全链路追溯报表配置

基于ARAS SQL Report引擎开发，支持导出PDF、Excel和Word格式。在ARAS Administration → Reports中创建（Report Type: SQL Report）。

📋 报表1：全链路追溯报表

用途：输入需求ID或技能ID，生成完整追溯报告，包含所有关联的Item信息。支持递归查询（WITH CTE）。
SQL全链路追溯报表 · WITH CTE递归查询
DECLARE @SourceId nvarchar(36) = '@SourceId' DECLARE @SourceType nvarchar(50) = '@SourceType' WITH TraceCTE AS ( -- 根节点：查询指定Item SELECT @SourceType AS ItemType, id AS ItemId, CASE @SourceType WHEN 'Digital Employee Requirement' THEN requirement_id WHEN 'Digital Employee Skill' THEN skill_id END AS ItemKey, name, status, version, NULL AS Relationship, NULL AS ParentId FROM [@SourceType] WHERE id = @SourceId UNION ALL -- 关联技能：需求→技能 SELECT 'Digital Employee Skill', s.id, s.skill_id, s.name, s.status, s.version, '实现', r.id FROM [Digital Employee Requirement] r JOIN [Requirement Skill] rs ON r.id = rs.source_id JOIN [Digital Employee Skill] s ON rs.related_id = s.id JOIN TraceCTE t ON r.id = t.ItemId UNION ALL -- 依赖技术组件：技能→组件 SELECT 'Atomic Technology Component', c.id, c.component_id, c.name, c.status, c.version, '依赖', s.id FROM [Digital Employee Skill] s JOIN [Skill Component] sc ON s.id = sc.source_id JOIN [Atomic Technology Component] c ON sc.related_id = c.id JOIN TraceCTE t ON s.id = t.ItemId ) SELECT * FROM TraceCTE

📈 报表2：技能开发进度报表

用途：展示所有技能的开发进度，按优先级和状态分组，用于项目管理和进度跟踪
SQL技能开发进度报表 · GROUP BY + STRING_AGG
SELECT priority AS 优先级, status AS 状态, COUNT(*) AS 数量, STRING_AGG(skill_id + ' - ' + name, '; ') AS 技能列表 FROM [Digital Employee Skill] GROUP BY priority, status ORDER BY CASE priority WHEN 'P0' THEN 1 WHEN 'P1' THEN 2 WHEN 'P2' THEN 3 END, CASE status WHEN '待开发' THEN 1 WHEN '开发中' THEN 2 WHEN '测试中' THEN 3 WHEN '运行中' THEN 4 WHEN '异常' THEN 5 END

📉 报表3：数字员工运行状态报表

用途：展示所有已上线数字员工的运行状态，按准确率降序
SQL运行状态报表 · WHERE status='运行中'
SELECT skill_id AS 技能ID, name AS 技能名称, version AS 版本号, status AS 状态, accuracy AS 运行准确率(%), call_count AS 累计调用次数, average_response_time AS 平均响应时间(ms), actual_online_date AS 上线时间 FROM [Digital Employee Skill] WHERE status = '运行中' ORDER BY accuracy DESC

✅ 报表4：合规审计报表

用途：生成开发过程合规审计报告，验证所有环节是否完整（测试通过率≥95%、部署记录完整）
SQL合规审计报表 · JOIN多表聚合
SELECT r.requirement_id AS 需求ID, r.name AS 需求名称, s.skill_id AS 技能ID, s.name AS 技能名称, s.version AS 版本号, s.actual_online_date AS 上线时间, COUNT(tc.id) AS 测试用例总数, SUM(CASE WHEN tc.last_execution_result = '通过' THEN 1 ELSE 0 END) AS 通过用例数, COUNT(d.id) AS 部署次数 FROM [Digital Employee Requirement] r JOIN [Requirement Skill] rs ON r.id = rs.source_id JOIN [Digital Employee Skill] s ON rs.related_id = s.id LEFT JOIN [Skill Test Case] stc ON s.id = stc.source_id LEFT JOIN [Test Case] tc ON stc.related_id = tc.id LEFT JOIN [Skill Deployment] sd ON s.id = sd.source_id LEFT JOIN [Deployment Record] d ON sd.related_id = d.id WHERE s.status = '运行中' GROUP BY r.requirement_id, r.name, s.skill_id, s.name, s.version, s.actual_online_date

🐍 核心Python同步脚本（sync_runtime_data.py）

通过REST API获取平台运行数据，更新到ARAS DigitalEmployeeSkill Item，包含异常告警功能。

PYsync_runtime_data.py · 主同步循环
import requests, json, time, logging from datetime import datetime # 配置日志 logging.basicConfig(level=logging.INFO, format='%(asctime)s - %(levelname)s - %(message)s') # 配置信息 ARAS_URL = "https://scsai-plm.yourcompany.com/innovator" PLATFORM_API_URL = "https://digital-employee.yourcompany.com/api/v1" WEBHOOK_URL = "your-webhook-url" # 企业微信/飞书 def get_aras_auth(): # ... 生成Basic Auth头(base64) auth_str = f"{ARAS_USERNAME}:{ARAS_PASSWORD}" return base64.b64encode(auth_str.encode()).decode() def get_running_skills(): resp = requests.get(f"{PLATFORM_API_URL}/skills?status=running", headers={"Authorization": f"Bearer {PLATFORM_API_TOKEN}"}) return resp.json() def update_aras_skill(skill_id, runtime_data): auth = get_aras_auth() # 1. 查询ARAS Item ID q = requests.get(f"{ARAS_URL}/server/odata/DigitalEmployeeSkill?$filter=skill_id eq '{skill_id}'", headers={"Authorization": f"Basic {auth}", "Accept": "application/json"}) item_id = q.json()["value"][0]["id"] # 2. 更新运行数据 requests.patch(f"{ARAS_URL}/server/odata/DigitalEmployeeSkill('{item_id}')", headers={"Authorization": f"Basic {auth}", "Content-Type": "application/json"}, json={"accuracy": runtime_data["accuracy"], "call_count": runtime_data["call_count"]}) logging.info(f"同步成功: {skill_id}") def send_alert(message): requests.post(WEBHOOK_URL, json={ "msgtype": "text", "text": {"content": f"【告警】{message}\n{datetime.now()}"}}) if __name__ == "__main__": while True: for skill in get_running_skills(): if skill["accuracy"] < 95: send_alert(f"准确率低于阈值: {skill['skill_id']}") update_aras_skill(skill["skill_id"], skill) time.sleep(300)
