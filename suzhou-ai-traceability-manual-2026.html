📋 需求管理模块

📥 提交新需求

  1. 点击左侧导航栏 "需求管理" → "提交需求"
  2. 填写需求信息：名称（≤20字）、描述、所属行业、优先级（P0/P1/P2）、期望上线时间、附件
  3. 点击"提交"，系统自动生成唯一需求ID（如 REQ-066）
  4. 系统自动将需求导入追溯表，并触发需求拆解智能体进行分析

📋 查看需求列表

  1. 点击 "需求管理" → "需求列表"
  2. 列表展示：需求ID、名称、优先级、状态、负责人、计划上线时间
  3. 支持按ID/名称/状态/优先级/时间进行筛选和搜索
  4. 点击任意需求ID，进入需求详情页面

✏️ 需求变更与关闭

  1. 在需求详情页面，点击"变更需求"按钮，可修改需求信息
  2. 系统自动记录变更历史，并同步更新追溯表和所有关联任务
  3. 所有技能都已上线并通过验收后，点击"关闭需求"，更新状态为"已关闭"
🧩 技能开发模块

📋 查看技能列表

  1. 点击 "技能开发" → "技能列表"
  2. 列表展示：技能ID、名称、所属智能体、优先级、状态、负责人、版本号、准确率
  3. 支持按ID/名称/状态/优先级/所属智能体筛选和搜索
  4. 点击任意技能ID，进入技能详情页面

🤖 方式一：从需求自动生成

  1. 在需求详情页面的"关联技能"标签页，点击"生成技能"按钮
  2. 系统根据追溯表自动生成标准化技能模板：技能ID/名称/描述、输入输出参数、依赖技术组件、验收标准
  3. 确认信息无误后，点击"创建"按钮

✏️ 方式二：手动创建

  1. 点击 "技能开发" → "创建技能"
  2. 填写技能基本信息：名称、描述、所属智能体、优先级
  3. 定义输入输出参数、选择依赖的原子技术组件、设置验收标准
  4. 点击"创建"，自动添加到追溯表

🎯 技能可视化编排

  1. 在技能详情页面，点击"技能编排"标签页
  2. 左侧为原子技术组件库，右侧为编排画布
  3. 从左侧拖拽原子技术组件到画布上，连接组件之间的数据流
  4. 配置每个组件的参数和运行条件
  5. 点击"保存"保存编排结果；点击"测试运行"在测试环境中验证功能

💻 代码管理

  1. 在技能详情页面，点击"代码管理"标签页
  2. 系统根据编排结果自动生成基础代码框架、API接口、单元测试
  3. 点击"克隆仓库"获取Git仓库地址，在本地进行开发
  4. 代码提交后，系统自动同步到平台，更新追溯表中的版本号和迭代记录
  5. 支持在线查看代码、对比不同版本的代码差异

📌 版本管理

  1. 在技能详情页面，点击"版本管理"标签页
  2. 展示所有历史版本：版本号、更新时间、更新内容、负责人
  3. 点击任意版本，可查看该版本的代码、编排结果、测试报告
  4. 支持版本回滚，点击"回滚到此版本"恢复到指定版本
🧪 测试管理模块

📋 测试用例管理

  1. 点击 "测试管理" → "测试用例"
  2. 系统根据追溯表自动生成所有技能的测试用例：ID、场景、前置条件、步骤、预期结果、验收标准
  3. 支持手动添加、编辑、删除测试用例
  4. 点击任意测试用例ID，进入用例详情页面

📝 创建测试任务

  1. 点击 "测试管理" → "创建测试任务"
  2. 选择需要测试的技能和测试用例
  3. 设置测试环境、测试时间、测试负责人
  4. 点击"创建"，系统自动分配任务并同步到Jira

🤖 自动化测试执行

  1. 在测试任务详情页面，点击"执行测试"
  2. 测试智能体自动执行所有测试用例，记录测试结果
  3. 测试完成后自动生成报告：通过/失败/跳过数、失败详情、性能结果
  4. 测试结果自动同步到追溯表，更新技能的测试状态

🐛 缺陷管理

  1. 测试中发现的缺陷自动记录到缺陷管理模块
  2. 每个缺陷自动关联到对应的需求、技能、代码模块和测试用例
  3. 支持手动添加缺陷，填写描述、严重程度、优先级
  4. 系统自动创建Jira工单，分配给对应负责人修复
  5. 缺陷修复后，自动触发回归测试
🚀 部署运维模块

🚀 一键部署

  1. 技能通过测试验收后，在技能详情页面点击"部署"按钮
  2. 选择部署环境：测试环境 / 预生产环境 / 生产环境
  3. 选择部署方式：云端部署 / 边缘部署
  4. 点击"确认部署"，系统自动执行部署流程

❓ 常见问题与解决方案
需求拆解不准确怎么办？▼

在需求详情页面点击"重新拆解"按钮，手动调整拆解结果；也可以联系平台管理员，优化需求拆解智能体的训练数据。

自动生成的代码不符合要求怎么办？▼

自动生成的代码只是基础框架，开发人员可以在本地进行修改和完善；也可以提交反馈，帮助平台优化代码生成智能体。

测试用例不完整怎么办？▼

支持手动添加和编辑测试用例；也可以将常用的测试用例保存为模板，方便后续复用。

部署失败怎么办？▼

查看部署日志，根据错误信息排查问题；常见问题包括：资源不足、镜像拉取失败、配置错误。

数字员工准确率下降怎么办？▼
  1. 查看运行数据，分析准确率下降的原因
  2. 收集新的样本数据，进行模型增量更新
  3. 调整模型参数或重新训练模型
  4. 更新追溯表中的版本号和准确率
