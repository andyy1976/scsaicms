SCSAI 运行数据同步与异常处理方案

ARAS PLM 运行时监控 · 自动同步 · 智能告警 · 安全回滚

📡 同步周期：每 5 分钟 🛡️ 自动回滚 🔔 多级告警 🚀 systemd 托管

1方案概述

目标：实现 SCSAI 运行时运行数据的 全自动化监控与同步，在检测到性能异常时无需人工干预，自动完成告警上报与变更单创建，保障苏州 AI 系统的 SLA 合规。

  • 每 5 分钟自动同步 — 运行数据（准确率、调用次数、响应时间、可用性）自动上报 ARAS PLM
  • 三项阈值监控 — 准确率 < 95%、平均响应时间 > 1000ms、可用性 < 99.5%
  • 异常自动处理 — 触发阈值自动创建变更单（ECR/ECO）、发送飞书告警
  • 自动回滚机制 — 连续 3 次触发阈值时自动回滚到上一个稳定版本
  • systemd 托管运行 — 服务稳定运行、断电自启、日志自动归档

2运行数据自动同步

运行脚本每 300 秒（5 分钟）自动执行，将以下四项核心 KPI 同步至 ARAS PLM Item / SCSAI_Runtime_Metrics：

指标名称 数据来源 ARAS 字段 正常范围
准确率 (Accuracy) metrics/accuracy.log accuracy_value ≥ 95%
调用次数 (API Calls) metrics/api_calls.json total_api_calls 动态统计
响应时间 (Response Time) metrics/response_time.log avg_response_ms ≤ 1000ms
可用性 (Availability) metrics/uptime.json availability_pct ≥ 99.5%

PYTHON /opt/scsaicloud/sync_aras_runtime.py 
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
# SCSAI Runtime Sync to ARAS PLM
# Author: SCSAI DevOps Team
# Version: 2.0 | Updated: 2026-05-10

import json
import time
import os
import sys
import logging
from datetime import datetime, timezone
from pathlib import Path

import requests
from aras_api import ArasConnector  # ARAS Innovator SDK
from feishu_sdk import FeishuBot    # 飞书 SDK

# ============ 配置区 ============
class Config:
    SYNC_INTERVAL   = 300          # 5 分钟同步一次（秒）
    ARAS_URL        = os.getenv("ARAS_URL",        "https://aras.scsaicloud.com")
    ARAS_DB         = os.getenv("ARAS_DB",         "SCSAI_Production")
    ARAS_USER       = os.getenv("ARAS_USER",       "sync_service")
    ARAS_PWD        = os.getenv("ARAS_PWD",        "")
    ITEM_TYPE       = "SCSAI_Runtime_Metrics"
    METRICS_DIR     = Path("/opt/scsaicloud/metrics")
    LOG_FILE        = Path("/var/log/scsaicloud/sync.log")
    FEISHU_WEBHOOK  = os.getenv("FEISHU_WEBHOOK", "")
    ENV             = os.getenv("ENV", "production")

    # 异常检测阈值
    THRESH_ACCURACY   = 95.0    # 准确率低于此值告警 (%)
    THRESH_RESPONSE   = 1000   # 响应时间高于此值告警 (ms)
    THRESH_AVAIL      = 99.5   # 可用性低于此值告警 (%)
    ROLLBACK_COUNT    = 3      # 连续触发 N 次自动回滚

# ============ 日志设置 ============
logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    handlers=[
        logging.FileHandler(Config.LOG_FILE, encoding="utf-8"),
        logging.StreamHandler(sys.stdout)
    ]
)
logger = logging.getLogger(__name__)


# ============ 数据采集 ============
class MetricsCollector:
    """从本地 metrics 目录采集运行时指标"""

    def collect(self) -> dict:
        ts = datetime.now(timezone.utc).isoformat()
        return {
            "timestamp":          ts,
            "accuracy":          self._read_float("accuracy.log",   0.0),
            "api_calls":          self._read_int("api_calls.json",   0),
            "avg_response_ms":     self._read_float("response_time.log", 0.0),
            "availability_pct":   self._read_float("uptime.json",      100.0),
        }

    def _read_float(self, filename: str, default: float) -> float:
        try:
            p = Config.METRICS_DIR / filename
            if filename.endswith(".json"):
                with open(p, encoding="utf-8") as f:
                    d = json.load(f)
                    return float(d.get("value", default))
            else:
                with open(p, encoding="utf-8") as f:
                    return float(f.read().strip())
        except Exception as e:
            logger.warning(f"[Collector] 读取 {filename} 失败: {e}, 使用默认值 {default}")
            return default

    def _read_int(self, filename: str, default: int) -> int:
        try:
            p = Config.METRICS_DIR / filename
            with open(p, encoding="utf-8") as f:
                d = json.load(f)
                return int(d.get("count", default))
        except Exception as e:
            logger.warning(f"[Collector] 读取 {filename} 失败: {e}")
            return default


# ============ ARAS PLM 同步 ============
class ArasSyncer:
    """将指标数据写入 ARAS Innovator"""

    def __init__(self):
        self.conn = ArasConnector(
            url=Config.ARAS_URL,
            db=Config.ARAS_DB,
            user=Config.ARAS_USER,
            pwd=Config.ARAS_PWD
        )
        logger.info("ARAS 连接已建立")

    def sync(self, metrics: dict) -> bool:
        """创建或更新 ARAS 中的运行时指标记录"""
        try:
            item = self.conn.new_item(Config.ITEM_TYPE, "add")
            item.set_property("metric_timestamp",   metrics["timestamp"])
            item.set_property("accuracy_value",     metrics["accuracy"])
            item.set_property("total_api_calls",    metrics["api_calls"])
            item.set_property("avg_response_ms",    metrics["avg_response_ms"])
            item.set_property("availability_pct",   metrics["availability_pct"])
            item.set_property("environment",        Config.ENV)
            item.set_property("sync_version",      "2.0")

            result = item.apply()
            item_id = result.get_id()
            logger.info(f"[ARAS] 同步成功 | ID={item_id} | 准确率={metrics['accuracy']}%")
            return True
        except Exception as e:
            logger.error(f"[ARAS] 同步失败: {e}")
            return False


# ============ 异常检测 ============
class AnomalyDetector:
    """检测运行数据是否触发阈值，触发时返回告警信息"""

    def check(self, metrics: dict) -> list:
        alerts = []
        if metrics["accuracy"] < Config.THRESH_ACCURACY:
            alerts.append({
                "type": "accuracy",
                "metric": "准确率",
                "value": metrics["accuracy"],
                "threshold": Config.THRESH_ACCURACY,
                "severity": "CRITICAL",
                "msg": f"准确率 {metrics['accuracy']}% < {Config.THRESH_ACCURACY}%（阈值）"
            })
        if metrics["avg_response_ms"] > Config.THRESH_RESPONSE:
            alerts.append({
                "type": "response",
                "metric": "平均响应时间",
                "value": metrics["avg_response_ms"],
                "threshold": Config.THRESH_RESPONSE,
                "severity": "CRITICAL",
                "msg": f"响应时间 {metrics['avg_response_ms']}ms > {Config.THRESH_RESPONSE}ms（阈值）"
            })
        if metrics["availability_pct"] < Config.THRESH_AVAIL:
            alerts.append({
                "type": "availability",
                "metric": "可用性",
                "value": metrics["availability_pct"],
                "threshold": Config.THRESH_AVAIL,
                "severity": "CRITICAL",
                "msg": f"可用性 {metrics['availability_pct']}% < {Config.THRESH_AVAIL}%（阈值）"
            })
        return alerts


# ============ 变更单处理 ============
class ChangeOrderCreator:
    """在 ARAS PLM 中创建变更单（ECR/ECO）"""

    def create_ecr(self, aras_conn, alert: dict, metrics: dict) -> str:
        try:
            ecr = aras_conn.new_item("ECR", "add")
            ecr.set_property("item_number",     "ECR-" + datetime.now().strftime("%Y%m%d%H%M%S"))
            ecr.set_property("description",    f"[自动] SCSAI 运行异常告警: {alert['msg']}")
            ecr.set_property("related_item",    "SCSAI_Runtime_Metrics")
            ecr.set_property("priority",        "1")
            ecr.set_property("classification",  "Runtime Anomaly")
            ecr.set_property("initiated_by",   "sync_service")
            result = ecr.apply()
            ecr_id = result.get_id()
            logger.info(f"[ECR] 创建成功 ID={ecr_id}")
            return ecr_id
        except Exception as e:
            logger.error(f"[ECR] 创建失败: {e}")
            return ""


# ============ 飞书告警 ============
class Feishualerter:
    """通过飞书 Webhook 发送告警消息"""

    def send(self, alerts: list, metrics: dict):
        if not Config.FEISHU_WEBHOOK:
            logger.warning("[Feishu] 未配置 Webhook，跳过告警")
            return
        if not alerts:
            return

        body = {
            "msg_type": "interactive",
            "card": {
                "header": {
                    "title": {"tag": "plain_text", "content": "🚨 SCSAI 运行异常告警"},
                    "template": "red"
                },
                "elements": [
                    {"tag": "div", "content": f"**时间：**{datetime.now().strftime('%Y-%m-%d %H:%M:%S')}　**环境：**{Config.ENV}"},
                    {"tag": "hr"},
                    *[{
                        "tag": "div",
                        "fields": [
                            {"is_short": True, "text": {"tag": "lark_md", "content": f"**告警项：**{a['metric']}"}},
                            {"is_short": True, "text": {"tag": "lark_md", "content": f"**当前值：**{a['value']}**（阈值：**{a['threshold']}**）"}}
                        ]
                    } for a in alerts],
                    {"tag": "hr"},
                    {"tag": "div", "content": "**操作：** 已自动创建 ARAS 变更单，请及时处理。"},
                    {
                        "tag": "action",
                        "actions": [{
                            "tag": "button",
                            "text": {"tag": "plain_text", "content": "🔗 前往 ARAS 处理"},
                            "type": "primary",
                            "url": f"{Config.ARAS_URL}"
                        }]
                    }
                ]
            }
        }
        try:
            r = requests.post(Config.FEISHU_WEBHOOK, json=body, timeout=10)
            r.raise_for_status()
            logger.info(f"[Feishu] 告警发送成功 ({r.status_code})")
        except Exception as e:
            logger.error(f"[Feishu] 告警发送失败: {e}")


# ============ 主程序 ============
def main():
    logger.info("=" * 50 + " SCSAI Runtime Sync 启动 " + "=" * 50)
    collector  = MetricsCollector()
    syncer     = ArasSyncer()
    detector   = AnomalyDetector()
    ecr_mgr    = ChangeOrderCreator()
    feishu_bot = Feishualerter()

    consecutive_alerts = 0

    while True:
        try:
            # ① 采集指标
            metrics = collector.collect()
            logger.info(f"[Loop] 采集指标: {metrics}")

            # ② 同步 ARAS
            sync_ok = syncer.sync(metrics)

            # ③ 异常检测
            alerts = detector.check(metrics)

            # ④ 处理告警
            if alerts:
                consecutive_alerts += 1
                logger.warning(f"[Alert] 检测到 {len(alerts)} 项异常（连续 {consecutive_alerts} 次）")
                for alert in alerts:
                    # 创建变更单
                    ecr_id = ecr_mgr.create_ecr(syncer.conn, alert, metrics)
                    alert["ecr_id"] = ecr_id
                # 飞书告警
                feishu_bot.send(alerts, metrics)

                # ⑤ 连续触发 → 自动回滚
                if consecutive_alerts >= Config.ROLLBACK_COUNT:
                    logger.critical(f"[Rollback] 连续 {consecutive_alerts} 次告警，触发自动回滚！")
                    trigger_rollback()
                    consecutive_alerts = 0  # 重置计数器
            else:
                consecutive_alerts = 0
                logger.info("[Loop] 指标正常，无告警")

        except Exception as e:
            logger.error(f"[Loop] 主循环异常: {e}", exc_info=True)

        # 等待下一个周期
        logger.info(f"[Loop] 等待 {Config.SYNC_INTERVAL}s...")
        time.sleep(Config.SYNC_INTERVAL)


def trigger_rollback():
    """触发自动回滚（详见第4节）"""
    import subprocess
    try:
        result = subprocess.run(
            ["/opt/scsaicloud/scripts/auto_rollback.sh"],
            capture_output=True, text=True, timeout=120
        )
        logger.critical(f"[Rollback] 回滚脚本输出: {result.stdout}")
        if result.returncode != 0:
            logger.error(f"[Rollback] 回滚脚本失败: {result.stderr}")
    except Exception as e:
        logger.error(f"[Rollback] 回滚执行异常: {e}")


if __name__ == "__main__":
    main()

3异常自动检测与处理

系统持续监控三项关键指标，任意一项触发阈值即视为异常，依次执行：ARAS 变更单创建 → 飞书告警推送 → 计数累计；连续触发 3 次自动执行回滚。

🚨 三项监控阈值（红色告警）

⚠️ 准确率告警
指标名称 Accuracy（准确率）
告警阈值 < 95.0%
检测方式 metrics/accuracy.log
ARAS 字段 accuracy_value
告警等级 🔴 CRITICAL
⚠️ 响应时间告警
指标名称 Response Time（响应时间）
告警阈值 > 1000 ms
检测方式 metrics/response_time.log
ARAS 字段 avg_response_ms
告警等级 🔴 CRITICAL
⚠️ 可用性告警
指标名称 Availability（可用性）
告警阈值 < 99.5%

告警等级 🔴 CRITICAL

处理流程图

异常等级定义

触发条件 等级 处理动作 通知范围
任意一项阈值首次触发 🔴 CRITICAL 创建 ECR + 飞书告警 + 计数器+1 运维团队 / 项目负责人
连续 2 次触发 🟡 WARNING 追加 ECR 记录 + 升级告警 运维团队 + 研发负责人
连续 3 次触发 🔴 BLOCK 自动回滚 + 发送紧急告警 全员通知（紧急通道）

4自动回滚机制

当异常计数器达到阈值（连续 3 次触发告警），系统自动执行回滚，无需人工介入，保障服务快速恢复。

回滚触发条件

  • 任意一项监控指标连续 3 个同步周期（15 分钟）均触发告警阈值
  • 上一次同步的 ARAS 指标记录中包含有效的上一版本标签（version_tag）
  • 回滚脚本执行权限验证通过（SSH Key / 服务账号）
  • 生产环境（ENV=production）自动执行；非生产环境仅发送告警不执行

自动回滚脚本（auto_rollback.sh）

BASH /opt/scsaicloud/scripts/auto_rollback.sh 
#!/usr/bin/env bash
# auto_rollback.sh — SCSAI 自动回滚脚本
# 由 sync_aras_runtime.py 调用，勿单独执行
# Author: SCSAI DevOps | V2.0 | 2026-05-10

set -euo pipefail

LOG_FILE="/var/log/scsaicloud/rollback.log"
ARAS_URL="${ARAS_URL:-https://aras.scsaicloud.com}"
SERVICE_NAME="scsai-inference-service"
DEPLOY_DIR="/opt/scsaicloud/deploy"
CONFIG_DIR="/opt/scsaicloud/config"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] [ROLLBACK] $*" | tee -a "$LOG_FILE"
}

check_env() {
    if [[ "$ENV" != "production" ]]; then
        log "非生产环境，仅发送告警，不执行回滚"
        exit 0
    fi
}

get_previous_version() {
    # 从 ARAS PLM 查询上一稳定版本标签
    log "从 ARAS 查询上一稳定版本..."
    PREV_TAG=$(curl -s -u "$ARAS_USER:$ARAS_PWD" \
        "$ARAS_URL/api/SCSAI_Versions?\$top=1&\$orderby=created_on desc" \
        | jq -r '.items[0].version_tag // empty')

    if [[ -z "$PREV_TAG" ]]; then
        log "ERROR: 未找到上一版本标签，回滚中止"
        exit 1
    fi
    log "上一版本: $PREV_TAG"
}

create_ecr() {
    # 在 ARAS 中创建回滚 ECR
    curl -s -u "$ARAS_USER:$ARAS_PWD" \
        -X POST "$ARAS_URL/api/ECR" \
        -H "Content-Type: application/json" \
        -d "$(cat <
    log "回滚变更单已创建"
}

send_alert() {
    # 发送紧急回滚告警
    curl -s -X POST "$FEISHU_WEBHOOK" \
        -H "Content-Type: application/json" \
        -d '{"msg_type":"text","content":{"text":"🚨 [紧急] SCSAI自动回滚已触发\n版本：'"$PREV_TAG"'\n时间：'"$(date '+%Y-%m-%d %H:%M:%S')"'\n请立即登录 ARAS PLM 确认！"}}'
}

rollback_deploy() {
    # 停止当前服务
    log "停止当前服务: $SERVICE_NAME"
    systemctl stop "$SERVICE_NAME" || log "服务停止失败，继续执行"

    # 切换到上一版本目录
    BACKUP_DIR="$DEPLOY_DIR/backup/$(date +%Y%m%d_%H%M%S)"
    log "备份当前版本到: $BACKUP_DIR"
    mkdir -p "$BACKUP_DIR"
    mv "$DEPLOY_DIR/current/"* "$BACKUP_DIR/" || true

    # 部署上一稳定版本
    PREV_PATH="$DEPLOY_DIR/versions/$PREV_TAG"
    if [[ -d "$PREV_PATH" ]]; then
        log "从 $PREV_PATH 部署上一版本"
        cp -r "$PREV_PATH/"* "$DEPLOY_DIR/current/"
    else
        log "ERROR: 版本目录不存在: $PREV_PATH"
        exit 1
    fi

    # 启动服务
    log "启动服务: $SERVICE_NAME"
    systemctl start "$SERVICE_NAME"
    sleep 10

    # 健康检查
    HEALTH=$(curl -sf http://localhost:8080/health || echo "fail")
    if [[ "$HEALTH" == "ok" ]]; then
        log "✅ 回滚成功，服务健康检查通过"
    else
        log "❌ 回滚后健康检查失败，请立即人工介入！"
        exit 1
    fi
}

# ============ 主流程 ============
log "========== 开始自动回滚 =========="
check_env
get_previous_version
create_ecr
send_alert
rollback_deploy
log "========== 回滚完成 =========="

回滚验证流程

步骤 验证内容 判定标准 失败处理
1. 服务状态 systemctl status 服务 Running active (running) 重新拉起服务，发送告警
2. HTTP 健康检查 GET /health 返回 {"status":"ok"} HTTP 200 + JSON 正确 自动告警，停止回滚流程
3. 指标验证 回滚后 3 个周期内指标回归正常 三项阈值全部恢复 追加人工检查工单
4. ARAS 同步确认 新版本标签已写入 ARAS Runtime Metrics version_tag 已更新 手动补录 ARAS 记录
5. 告警恢复确认 飞书告警消息更新为「已恢复」 告警通道静默 人工确认后关闭告警

6监控与告警配置

除脚本内置的飞书告警外，建议额外配置以下监控手段，确保告警无死角覆盖。

告警通道矩阵

告警类型 触发条件 告警渠道 接收人
⚠️ 阈值触发（首次） 任意一项指标首次触发阈值 飞书 Webhook（普通） 运维团队
⚠️ 连续告警 同一指标连续 2 次触发 飞书 Webhook（升级）+ 短信 运维 + 研发负责人
🔴 回滚触发 连续 3 次触发（自动回滚） 飞书（紧急） + 电话 + 短信 全员紧急通道
🟠 服务异常 systemd 服务状态 != Running 飞书 + Prometheus AlertManager 运维团队
🟠 同步失败 ARAS API 返回非 200 飞书 Webhook 运维团队

Prometheus 监控指标（可选）

PROMETHEUS CONFIG 
# prometheus 告警规则 — aras_runtime_sync.yml
groups:
  - name: aras-runtime-sync
    interval: 30s
    rules:
      - alert: ARASTransactionFailed
        expr: aras_sync_errors_total > 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "ARAS 同步失败"
          description: "过去 5 分钟内有 {{ $value }} 次同步失败"

      - alert: SCSAIAvailabilityLow
        expr: scsai_availability_pct < 99.5
        for: 3m
        labels:
          severity: warning
        annotations:
          summary: "SCSAI 可用性低于阈值"
          description: "可用性 {{ $value }}% < 99.5%"

      - alert: SCSAIAccuracyLow
        expr: scsai_accuracy_value < 95
        for: 3m
        labels:
          severity: warning
        annotations:
          summary: "SCSAI 准确率低于阈值"
          description: "准确率 {{ $value }}% < 95%"

      - alert: SCSAIResponseSlow
        expr: scsai_avg_response_ms > 1000
        for: 3m
        labels:
          severity: warning
        annotations:
          summary: "SCSAI 响应时间超标"
          description: "平均响应时间 {{ $value }}ms > 1000ms"

      - alert: SyncServiceDown
        expr: up{job="aras-runtime-sync"} == 0
        for: 1m
        labels:
          severity: critical
        annotations:
          summary: "同步服务已停止"
          description: "aras-runtime-sync 服务已停止超过 1 分钟"

7常见问题与解决方案

Q1: 同步脚本启动失败，提示 ModuleNotFoundError: No module named 'aras_api'？+
安装 ARAS Innovator Python SDK：pip3 install aras-api
或确认 SDK 路径在 PYTHONPATH 中。如使用虚拟环境，确认已激活正确的 venv。
Q2: 飞书告警收不到，但日志显示告警已发送？+
检查 Webhook 地址是否正确、机器人是否已在群中添加、群机器人是否被禁用。Webhook 有每日调用上限（普通机器人 500次/分钟），确认未超限。
Q3: ARAS PLM 连接超时，频繁同步失败？+
检查网络连通性：ping aras.scsaicloud.com；确认防火墙规则允许出站 HTTPS（443）；检查 ARAS 服务端是否在维护窗口期。可临时增加 timeout 参数至 30s。
Q4: 误触发自动回滚，如何快速恢复？+
① 立即停止回滚链：systemctl stop aras-runtime-sync
② 手动重新部署目标版本：bash /opt/scsaicloud/scripts/deploy.sh [version_tag]
③ 重置告警计数器（在 ARAS 中将相关 ECR 状态改为 Closed）
④ 恢复服务：systemctl start aras-runtime-sync
Q5: systemd 服务自动重启次数耗尽怎么办？+
先排查根因（journalctl -u aras-runtime-sync -b 查看崩溃日志），修复问题后执行：systemctl reset-failed aras-runtime-sync，再重新启动服务。
Q6: 如何临时跳过告警（节假日维护期间）？+
创建维护窗口文件：touch /opt/scsaicloud/.maintenance_mode
脚本会自动检测此文件并跳过告警发送（同步仍正常进行）；维护结束后删除该文件即可。
Q7: 回滚后旧版本目录被覆盖，无法再次回滚？+
部署时务必保留历史版本：确保 /opt/scsaicloud/versions/ 目录至少有最近 3 个版本的完整快照；CI/CD 流水线应自动归档历史构建产物。
Q8: 指标文件不存在导致采集失败？+
脚本会使用默认值（accuracy=0.0, availability=100.0）防止中断；长期解决方案：确保 SCSAI 推理服务在运行时持续写入 /opt/scsaicloud/metrics/ 目录下的指标文件。
