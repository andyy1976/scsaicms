🔄 ARAS API版GitLab CI自动化流水线

保存为 .gitlab-ci.yml 放在代码仓库根目录，配置好环境变量后即可实现全流程自动化。

1

🚀 初始化 (init)

从ARAS获取技能信息，计算新版本号，更新状态为"构建中"，添加评论

GET /server/odata/DigitalEmployeeSkill?$filter=skill_id eq '...'
2

🔍 代码检查 (code-check)

pylint + flake8质量检查，上传报告到数字员工平台

POST /skills/$SKILL_ID/code-quality
3

🧪 单元测试 (unit-test)

pytest执行，生成覆盖率报告和JUnit XML，上传到平台

POST /skills/$SKILL_ID/unit-test
4

🐳 构建镜像 (build)

构建Docker镜像并推送到仓库，更新ARAS中镜像信息

PATCH /DigitalEmployeeSkill('$ID') → docker_image
5

🧪 部署测试环境 (deploy-test)

调用平台API部署到测试环境，健康检查，失败则更新ARAS状态为"异常"

POST /skills/$SKILL_ID/deploy → environment: test
6

🔗 集成测试 (integration-test)

调用平台API执行集成测试，通过率≥95%才能继续

POST /skills/$SKILL_ID/integration-test
7

✅ 验收测试 (acceptance-test)

自动化验收测试，通过后更新ARAS状态为"待部署"

PATCH /DigitalEmployeeSkill → status: 待部署
8

🚀 部署生产 (deploy-prod)

灰度10%部署→健康检查→创建Deployment Record→更新状态"运行中"

POST /DeploymentRecord → gray_percentage: 10
9

🎉 完成 (finish)

创建版本基线，添加完成评论，所有信息同步到ARAS PLM

POST /CreateBaseline → 技能发布基线

📋 核心ARAS API交互点

YAML.gitlab-ci.yml · init阶段核心逻辑
# 生成ARAS Basic Auth头 ARAS_AUTH=$(echo -n "$ARAS_USERNAME:$ARAS_PASSWORD" | base64 -w 0) # 从ARAS获取技能信息 response=$(curl -s -H "Authorization: Basic $ARAS_AUTH" \ -H "Accept: application/json" \ "$ARAS_URL/server/odata/DigitalEmployeeSkill?\$filter=skill_id eq '$SKILL_ID'") # 解析返回 SKILL_NAME=$(echo $response | jq -r '.value[0].name') CODE_MODULE_PATH=$(echo $response | jq -r '.value[0].code_module_path') CURRENT_VERSION=$(echo $response | jq -r '.value[0].version') # 计算新版本号 NEW_VERSION=$(echo $CURRENT_VERSION | awk -F. '{print $1"."$2"."($3+1)}') # 更新ARAS状态为"构建中" curl -s -X PATCH -H "Authorization: Basic $ARAS_AUTH" \ -H "Content-Type: application/json" \ "$ARAS_URL/server/odata/DigitalEmployeeSkill('$ARAS_ITEM_ID')" \ -d '{"status":"构建中","build_url":"$CI_JOB_URL"}' # 在ARAS中添加评论 curl -s -X POST -H "Authorization: Basic $ARAS_AUTH" \ -H "Content-Type: application/json" \ "$ARAS_URL/server/odata/Comments" \ -d '{"source_id":"$ARAS_ITEM_ID","comments":"🤖 流水线已启动 v$NEW_VERSION"}'

🚀 生产部署灰度策略

YAML.gitlab-ci.yml · deploy-prod阶段
# 调用平台API部署（默认灰度10%） response=$(curl -s -X POST -H "Authorization: Bearer $PLATFORM_API_TOKEN" \ "$PLATFORM_API_URL/skills/$SKILL_ID/deploy" \ -d '{"version":"$NEW_VERSION","environment":"prod", "gray_release":true,"gray_percentage":10}') # 健康检查 response=$(curl -s "$PLATFORM_API_URL/skills/$SKILL_ID/health?environment=prod") if [ "$(echo $response | jq -r '.status')" != "healthy" ]; then # 更新ARAS状态为异常 curl -s -X PATCH ... -d '{"status":"异常"}' exit 1 fi # 在ARAS中创建部署记录 curl -s -X POST ... "$ARAS_URL/server/odata/DeploymentRecord" \ -d '{"deployment_id":"$UUID","skill_id":"$SKILL_ID", "version":"$NEW_VERSION","environment":"生产环境", "deploy_status":"成功","gray_percentage":10}' # 更新ARAS状态为"运行中" curl -s -X PATCH ... -d '{"version":"$NEW_VERSION","status":"运行中"}'

🚀 部署与配置指南

1️⃣ 环境变量配置

在GitLab CI/CD设置中添加以下保护变量：

🔐 ARAS PLM

  • ARAS_USERNAME = automation-bot
  • ARAS_PASSWORD = 自动化机器人密码

🌐 数字员工平台

  • PLATFORM_API_TOKEN = 平台API访问令牌

🐳 Docker仓库

  • DOCKER_REGISTRY_USER = 仓库用户名
  • DOCKER_REGISTRY_PASSWORD = 仓库密码

2️⃣ ARAS导入步骤

  1. 登录ARAS Innovator管理员界面
  2. 进入 Administration → Import
  3. 选择 DigitalEmployeeTraceability.xml 文件
  4. 点击 Import 按钮，等待导入完成
  5. 验证6个ItemType是否都已成功创建
  6. 重置 automation-bot 账号的密码
